    row_count = len(table)
    col_count = max(len(row) for row in table)
    return row_count >= 2 and col_count >= 2

## 8. 代码层面对比

### Dify - Extractor 模式
```python
# dify/api/core/rag/extractor/extract_processor.py
class ExtractProcessor:
    @staticmethod
    def load_file_type(file_type: str):
        # 根据文件类型选择提取器
        extractor_class = extractor_factory.get_extractor(file_type)
        return extractor_class.extract()
```

### RAGFlow - DeepDoc 模式
```python
# ragflow/deepdoc/
class DeepDocParser:
    def parse(self, file_path):
        # 1. 布局分析
        layout = self.layout_analysis(file_path)
        
        # 2. OCR 识别
        ocr_result = self.ocr_recognition(layout)
        
        # 3. 内容重组
        content = self.reconstruct(ocr_result)
        
        return content
```

### Coze Studio - Parser Factory 模式
```python
# coze-studio/backend/infra/document/parser/parser.py
class Parser:
    def parse_file(self, file_name, file_type, content):
        # 根据文件类型获取解析器
        cls = self.registry.get_parser_class(engine, file_type)
        parser = cls(file_name=file_name, file_type=file_type)
        return parser.parse(content)
```

### WeKnora - Registry 模式
```python
# WeKnora/docreader/parser/parser.py
class Parser:
    def __init__(self):
        self.registry = registry
    
    def parse_file(self, file_name, file_type, content):
        cls = self.registry.get_parser_class("", file_type)
        parser = cls(file_name=file_name, file_type=file_type)
        return parser.parse(content)
```

### 各平台文档解析特点总结:

**Dify:**
- 格式支持最全面
- 支持 Notion、Confluence 等第三方集成
- Firecrawl/WaterCrawl 双 Web 爬取方案

**RAGFlow:**
- DeepDoc 深度文档理解
- 布局分析能力最强
- 适合复杂 PDF 处理

**QAnything:**
- 专注于基础文档解析
- 与检索模块深度集成
- 简洁高效

**Coze Studio:**
- 轻量级设计
- 基于 pdfplumber 的 PDF 解析
- 适合基础文档处理

**WeKnora:**
- Registry 模式设计优雅
- 支持多种解析引擎
- 代码结构清晰

**FastGPT:**
- 文档处理与工作流深度集成
- 支持常见格式
- 易用性好
