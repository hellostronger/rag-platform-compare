# 功能特性差异对比

## 1. 平台概述与架构

### 1.1 Dify
- **定位**: 一站式 AI 应用开发平台
- **技术栈**: Python (Flask) + TypeScript (React)
- **架构**: 微服务架构，支持分布式部署
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/langgenius/dify

### 1.2 RAGFlow
- **定位**: 基于深度文档理解的 RAG 引擎
- **技术栈**: Python (Flask) + TypeScript (React)
- **架构**: 微服务架构 + DeepDoc 文档处理引擎
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/infiniflow/ragflow

### 1.3 QAnything
- **定位**: 一站式检索问答服务
- **技术栈**: Python + React
- **架构**: 前后端分离架构
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/netease-youdao/QAnything

### 1.4 Coze Studio
- **定位**: 一站式 AI 助手开发平台
- **技术栈**: Go + TypeScript (React)
- **架构**: DDD 微服务架构，使用 Rush.js 管理 monorepo
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/coze-dev/coze-studio

### 1.5 WeKnora
- **定位**: 企业级 RAG 解决方案
- **技术栈**: Python + TypeScript
- **架构**: 微服务架构
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/Tencent/WeKnora

### 1.6 FastGPT
- **定位**: 可视化 AI 应用构建平台
- **技术栈**: TypeScript (Next.js) + Python
- **架构**: Next.js + 插件系统
- **开源协议**: Apache 2.0
- **GitHub**: https://github.com/labring/FastGPT

class ParentChildIndexProcessor(BaseIndexProcessor):
    def transform(self, documents, current_user=None, **kwargs):
        rules = Rule.model_validate(process_rule.get("rules"))
        if rules.parent_mode == ParentMode.PARAGRAPH:
            # 段落模式：先分段再创建子节点
            document_nodes = splitter.split_documents([document])
            for document_node in document_nodes:
                child_nodes = self._split_child_nodes(document_node, rules, ...)
                document_node.children = child_nodes
        elif rules.parent_mode == ParentMode.FULL_DOC:
            # 全文模式：整个文档作为父块
            page_content = "\n".join([document.page_content for document in documents])
            child_nodes = self._split_child_nodes(document, rules, ...)
            document.children = child_nodes
    
    def load(self, dataset, documents, ...):
        # 只为子节点创建向量索引
        for document in documents:
            child_documents = document.children
            if child_documents:
                vector.create(formatted_child_documents)

```python
class SelfParentRetriever(ParentDocumentRetriever):
    async def _aget_relevant_documents(self, query, run_manager):
        # 1. 向量检索获取子文档
        if self.search_type == "mmr":
            sub_docs = await self.vectorstore.amax_marginal_relevance_search(...)
        else:
            res = await self.vectorstore.asimilarity_search_with_score(...)
            scores = [score for _, score in res]
            sub_docs = [doc for doc, _ in res]
        
        # 2. 获取父文档 ID
        ids = []
        for d in sub_docs:
            if self.id_key in d.metadata:
                ids.append(d.metadata[self.id_key])
        
        # 3. 从文档存储获取父文档
        docs = await self.docstore.amget(ids)
        
        # 4. 合并分数到父文档
        for i, doc in enumerate(docs):
            if doc is not None:
                doc.metadata['score'] = scores[i]
        
        return [d for d in docs if d is not None]
    
    async def aadd_documents(self, documents, ids=None, ...):
        # 1. 父文档分割
        if self.parent_splitter is not None:
            split_documents = []
            for doc in documents:
                if doc.metadata['has_table'] or num_tokens_embed(doc.page_content) <= parent_chunk_size:
                    split_documents.append(doc)
                else:
                    need_split_docs.append(doc)
            split_documents.extend(self.parent_splitter.split_documents(need_split_docs))
        
        # 2. 子文档分割
        for doc in documents:
            sub_docs = self.child_splitter.split_documents([doc])
            for _doc in sub_docs:
                _doc.metadata[self.id_key] = _id  # 关联父文档 ID
            docs.extend(sub_docs)
        
        # 3. 创建向量索引 (子文档)
        await self.vectorstore.aadd_documents(embed_docs)
        
        # 4. 存储父文档到文档存储
        if add_to_docstore:
            await self.docstore.amset(full_docs)

class Vector:
    def __init__(self, dataset: Dataset, attributes: list | None = None):
        self._dataset = dataset
        self._embeddings = self._get_embeddings()
        self._attributes = attributes
        self._vector_processor = self._init_vector()
    
    def _init_vector(self) -> BaseVector:
        vector_type = dify_config.VECTOR_STORE
        
        if self._dataset.index_struct_dict:
            vector_type = self._dataset.index_struct_dict["type"]
        
        vector_factory_cls = self.get_vector_factory(vector_type)
        return vector_factory_cls().init_vector(self._dataset, self._attributes, self._embeddings)
```

## 7. 关键差异总结

### 7.1 各平台核心优势

| 平台 | 核心优势 | 适合场景 |
|------|---------|---------|
| Dify | 索引处理器丰富、支持 10+ 向量后端 | 企业级应用、多数据源集成 |
| RAGFlow | DeepDoc 深度文档理解、GraphRAG | 复杂文档处理、知识图谱 |
| QAnything | 父子文档检索简洁高效 | 快速部署的检索问答 |
| Coze Studio | DDD 架构、企业级稳定性 | 规模化企业应用 |
| WeKnora | 文档处理链完整 | 企业文档管理 |
| FastGPT | 可视化工作流、插件丰富 | 快速原型开发 |

### 7.2 技术选型建议

- **复杂 PDF 解析**: RAGFlow (DeepDoc)
- **多向量后端支持**: Dify
- **简单快速部署**: QAnything
- **知识图谱增强**: RAGFlow
- **可视化开发**: FastGPT
- **企业级架构**: Coze Studio
