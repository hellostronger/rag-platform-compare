# 检索流程差异对比

## 1. 检索流程架构对比

### 1.1 Dify 检索流程

```
用户查询
  │
  ▼
查询预处理 (分词、向量化)
  │
  ├───► 关键词检索 (全文/倒排索引)
  │
  ├───► 向量检索 (语义相似度)
  │
  ▼
多路召回结果合并
  │
  ▼
Rerank 重排序 (可选)
  │
  ▼
分数阈值过滤
  │
  ▼
Top-K 筛选
  │
  ▼
后处理 (去重、格式化)
  │
  ▼
返回检索结果
```

**核心代码位置:**
- `dify/api/core/rag/retrieval/dataset_retrieval.py` - 主检索逻辑
- `dify/api/core/rag/datasource/retrieval_service.py` - 检索服务
- `dify/api/core/rag/index_processor/` - 索引处理器

**关键特性:**
- 支持多种检索方式：语义检索、全文检索、混合检索
- 多数据集路由：React 路由和函数调用路由
- 支持父子文档结构检索
- 完善的元数据过滤机制
- 支持多种向量数据库后端

### 1.2 RAGFlow 检索流程

```
用户查询
  │
  ▼
查询预处理
  │
  ├───► 元数据过滤 (自动/手动)
  │
  ├───► 跨语言处理 (可选)
  │
  ▼
检索执行
  │
  ├───► 向量检索 (Milvus/Infinity)
  │
  ├───► 关键词检索
  │
  ▼
混合检索合并
  │
  ▼
Rerank 重排序
  │
  ├───► TOC 增强检索 (可选)
  │
  ├───► KG 知识图谱检索 (可选)
  │
  ▼
子块检索 (Child Chunk)
  │
  ▼
返回结果
```

**核心代码位置:**
- `ragflow/agent/tools/retrieval.py` - 检索工具
- `ragflow/rag/nlp/search.py` - 搜索核心逻辑
- `ragflow/rag/graphrag/search.py` - GraphRAG 检索

**关键特性:**
- 支持按数据集名称动态检索
- 元数据过滤支持自动和手动模式
- TOC (Table of Contents) 增强检索
- KG (Knowledge Graph) 图谱检索增强
- 跨语言检索支持

### 1.3 QAnything 检索流程

```

  │
  ▼
查询向量化
  │
  ├───► Milvus 向量检索 (带分区过滤)
  │
  ├───► ES 关键词检索 (混合检索时)
  │
  ▼
ParentDocumentRetriever 处理
  │
  ├───► 子块检索
  │
  ├───► 父文档还原
  │
  ▼
Rerank 重排序
  │
  ▼
返回检索结果
```

**核心代码位置:**
- `qanything_kernel/core/retriever/parent_retriever.py` - 父检索器
- `qanything_kernel/core/retriever/vectorstore.py` - 向量存储
- `qanything_kernel/core/retriever/elasticsearchstore.py` - ES 存储

**关键特性:**
- ParentDocumentRetriever 实现父子文档检索
- Milvus 分区检索
- ES 混合检索支持
- 可配置的父子块大小

## 3. 代码层面差异

### Dify
```python
# dify/api/core/rag/retrieval/dataset_retrieval.py
def knowledge_retrieval(self, request: KnowledgeRetrievalRequest) -> list[Source]:
    # 1. 获取可用数据集
    available_datasets = self._get_available_datasets(...)
    
    # 2. 元数据过滤
    metadata_filter_document_ids, metadata_condition = self.get_metadata_filter_condition(...)
    
    # 3. 检索执行
    documents = self.retrieval_service.retrieve(...)
    
    # 4. 后处理
    return self.process_results(documents)
```

### RAGFlow
```python
# ragflow/agent/tools/retrieval.py
async def _retrieve_kb(self, query_text: str):
    # 1. 解析查询
    query = self.string_format(query_text, vars)
    
    # 2. 元数据过滤
    doc_ids = await apply_meta_data_filter(...)
    
    # 3. 跨语言处理
    if self._param.cross_languages:
        query = await cross_languages(...)
    
    # 4. 检索执行
    kbinfos = await settings.retriever.retrieval(...)
    
    # 5. TOC 增强
    if self._param.toc_enhance:
        cks = await settings.retriever.retrieval_by_toc(...)
```

### QAnything
```python
# qanything_kernel/core/retriever/parent_retriever.py
async def get_retrieved_documents(self, query, partition_keys, time_record, hybrid_search, top_k):
    # 1. 设置检索条件
    expr = f'kb_id in {partition_keys}'
    self.retriever.set_search_kwargs("similarity", k=top_k, expr=expr)
    
    # 2. Milvus 检索
    query_docs = await self.retriever.aget_relevant_documents(query)
    
    # 3. ES 混合检索 (可选)
    if hybrid_search:
        es_docs = await self.es_store.asimilarity_search(...)
    
    return query_docs
```
