# 索引差异对比

## 1. 索引架构对比

### 1.1 Dify 索引架构

```
索引系统
├── IndexProcessor (索引处理器)
│   ├── IndexProcessorFactory - 工厂类
│   ├── IndexProcessorBase - 基类
│   └── Processor Implementations
│       ├── ParagraphIndexProcessor - 段落索引
│       ├── ParentChildIndexProcessor - 父子索引
│       └── QAIndexProcessor - 问答索引
│
├── Index Structure Types
│   ├── paragraph - 段落结构
│   ├── parent_child - 父子结构
│   ├── qa - 问答结构
│   └── automatic - 自动结构
│
├── Vector Index (向量索引)
│   ├── Vector Factory
│   ├── Vector Store Interfaces
│   └── Backends (Milvus, Qdrant, Weaviate, etc.)
│
├── Keyword Index (关键词索引)
│   ├── JiebaKeywordTableHandler
│   ├── ElasticSearchHandler
│   └── KeywordTableStore
│
└── Metadata Index (元数据索引)
    ├── DatasetMetadata
    └── MetadataFiltering
```

**核心文件:**
- `dify/api/core/rag/index_processor/index_processor.py`
- `dify/api/core/rag/index_processor/processor/paragraph_index_processor.py`
- `dify/api/core/rag/index_processor/processor/parent_child_index_processor.py`
- `dify/api/core/rag/index_processor/processor/qa_index_processor.py`

### 1.2 RAGFlow 索引架构

```
索引系统
├── Index Builder
│   ├── Chunking Engine - 分块引擎
│   ├── Parser Factory - 解析器工厂
│   └── Tokenizer - 分词器
│
├── Vector Index
│   ├── Milvus Index
│   ├── Infinity Index
│   └── Index Builder
│
├── Full-Text Index (全文索引)
│   ├── Elasticsearch Index
│   └── Inverted Index
│
├── Graph Index (图谱索引)
│   ├── Knowledge Graph Builder
│   ├── Entity Extraction
│   └── Relation Extraction
│
└── TOC Index (目录索引)
    ├── Table of Contents
    └── Hierarchy Index
```

**核心文件:**
- `ragflow/rag/nlp/search.py`
- `ragflow/rag/graphrag/search.py`
- `ragflow/deepdoc/` - 文档处理模块

### 1.3 QAnything 索引架构

```
索引系统
├── ParentDocumentRetriever
│   ├── Vector Store (Milvus)
│   │   ├── Parent Chunks
│   │   └── Child Chunks
│   │
│   ├── Document Store (MySQL)
│   │   └── Full Documents
│   │
│   └── Keyword Store (Elasticsearch)
│       └── Full-text Index
│
└── Index Configuration
    ├── DEFAULT_PARENT_CHUNK_SIZE
    ├── DEFAULT_CHILD_CHUNK_SIZE
    └── SEPARATORS
```

**核心文件:**
- `qanything_kernel/core/retriever/parent_retriever.py`
- `qanything_kernel/core/retriever/docstore.py`
- `qanything_kernel/configs/model_config.py`

### 1.4 Coze Studio 索引架构

```
索引系统
├── Knowledge Base Index
│   ├── Vector Index (Milvus)
│   ├── Full-text Index (Elasticsearch)
│   └── Metadata Index (MySQL)
│
└── Index Configuration
    └── Model Config
```

### 1.5 WeKnora 索引架构

```
索引系统
├── Document Index
│   ├── Vector Store
│   └── Document Store
│
└── QA Dataset Index
    └── QA Pairs
```

### 1.6 FastGPT 索引架构

```
索引系统
├── Dataset Index
│   ├── Vector Index
│   ├── Full-text Index
│   └── Metadata Index
│
└── Chunk Index
    ├── Parent Chunk
    └── Child Chunk
```

## 3. 索引构建流程对比

### 3.1 Dify 索引构建流程

```
文档上传
  │
  ▼
文档解析 (Extractor)
  │
  ▼
文本分块 (Splitter)
  │
  ├───► 段落索引处理器
  │     ├── 创建向量索引
  │     └── 创建关键词索引
  │
  ├───► 父子索引处理器
  │     ├── 创建父块
  │     ├── 创建子块
  │     ├── 向量索引 (子块)
  │     └── 文档存储 (父块)
  │
  └───► QA 索引处理器
        ├── 生成 QA 对
        └── 创建索引
  │
  ▼
索引持久化
  │
  ▼
完成
```

**代码示例:**
```python
# dify/api/core/rag/index_processor/processor/paragraph_index_processor.py
class ParagraphIndexProcessor(IndexProcessorBase):
    def index(self, dataset, document, chunks):
        # 1. 创建向量索引
        self._index_vector(dataset, chunks)
        
        # 2. 创建关键词索引
        self._index_keyword(dataset, chunks)
        
        # 3. 保存文档段
        self._save_segments(dataset, document, chunks)
```

### 3.2 RAGFlow 索引构建流程

```
文档上传
  │
  ▼
DeepDoc 解析
  │
  ├───► 布局分析
  ├───► OCR 识别
  └───► 表格提取
  │
  ▼
智能分块
  │
  ├───► 向量化
  ├───► 关键词提取
  └───► 图谱构建
  │
  ▼
索引存储
  │
  ▼
完成
```

### 3.3 QAnything 索引构建流程

```
文档上传
  │
  ▼
文档解析
  │
  ▼
Parent Splitter (512 tokens)
  │
  ▼
Child Splitter (128 tokens, 25% overlap)
  │
  ├───► Milvus (子块向量)
  │
  ├───► MySQL (父块全文)
  │
  └───► ES (关键词索引)
  │
  ▼
完成
```

**代码示例:**
```python
# qanything_kernel/core/retriever/parent_retriever.py
async def aadd_documents(self, documents, ids, add_to_docstore, parent_chunk_size, ...):
    # 1. 父块分割
    if self.parent_splitter is not None:
        documents = self.parent_splitter.split_documents(documents)
    
    # 2. 子块分割
    for doc in documents:
        sub_docs = self.child_splitter.split_documents([doc])
        docs.extend(sub_docs)
    
    # 3. 向量索引 (子块)
    res = await self.vectorstore.aadd_documents(docs)
    
    # 4. 文档存储 (父块)
    if add_to_docstore:
        await self.docstore.amset(full_docs)
```
